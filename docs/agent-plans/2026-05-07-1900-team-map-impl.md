## 1. Task summary

Реализовать `/sandbox/team-map` — preview-surface с пространственной визуализацией handoff'ов между агентами (Pentagon-style, не Paperclip-style). Узлы = карточки агентов из fixtures, рёбра = handoff'ы за окно времени, hover = summary, click = navigate в `/activity/:runId`.

Mock-only. Backend этого поведения не содержит. Помечено `<MockBadge kind="design">`. Sidebar entry — в sandbox-секции под divider'ом, рядом с `team-bridge`.

Фича изолирована: удаляется одной операцией (route + screen + sidebar item + handoffs из fixtures + types + § 1.16 в backend-gaps).

## 4. Decisions on open questions (design doc §10)

Принимаю решения, чтобы не блокировать impl. Любое можно перепродать одним аргументом.

| # | Q | Решение |
|---|---|---|
| 1 | Дефолт окна | **7d.** 24h рискует empty state на demo data, 30d начнёт «месить» граф. 7d балансит. |
| 2 | Глагол | **`asked`.** Универсально, читается как обычная просьба между сотрудниками. Если зайдёт hover-тест с заказчиком — попробуем `passed to` для делегирующих случаев. |
| 3 | Pending → timed_out порог | **4 часа.** В seed'ах ставлю явный `status: 'timed_out'` где это релевантно — клиент ничего не пересчитывает. UI рендерит как есть. |
| 4 | Layout | **Static positions** в fixtures (`agentPositions: Record<agent_id, {x, y}>`). Force-directed на 2-4 узлах = хаос, на demo не нужно. |
| 5 | Cluster'ы | **Out of scope.** |
| 6 | Approval ↔ handoff | **Out of scope для preview.** Seed'ы не трогают external actions. |
| 7 | Cross-workspace | Filter cascade применяется. `listHandoffs` фильтрует через `inCurrentWorkspace(from_agent_id)`. |
| 8 | Dataset | **10 handoffs в ws_ops** (рассыпаны по 7 дням, разные статусы). 0 в ws_growth (1 агент — empty case). 0 в ws_finance (1 агент — empty case). |

## 5. Domain model changes (§6 design doc, скорректировано)

### 5.1 `RunStepType` extension (`types.ts:202`)

```ts
export type RunStepType =
  | 'llm_call'
  | 'tool_call'
  | 'memory_read'
  | 'memory_write'
  | 'approval_gate'
  | 'validation'
  | 'agent_call'   // ← новый, mock-only
```

**Note:** `agent_call` пока никем не эмитится в существующих run fixtures. Добавляю чисто как future-proofing. Если `stepTypeToAudit` в api.ts требует exhaustive map — добавлю явный mapping `'agent_call' → 'agent_call'` (audit timeline толерантен к free-form `step_type` per types.ts:344-352).

### 5.2 Новая сущность `Handoff`

```ts
export type HandoffStatus = 'pending' | 'answered' | 'timed_out' | 'declined'

export interface Handoff {
  id: string
  run_id: string                 // в каком run возник; для navigation на /activity/:runId
  from_agent_id: string
  to_agent_id: string
  summary: string                // human-readable одна строка ("is there an open contract?")
  status: HandoffStatus
  created_at: string
  resolved_at: string | null
  // mock-only denorm для filter cascade. Реальный backend выводил бы из
  // agent.workspace_id (которого нет в spec). Фиксируется в § 1.16.
  workspace_id: string
}

export interface HandoffList {
  items: Handoff[]
  total: number
}
```

**Изменения относительно design doc §6.2:**
- `resolved_at: string | null` (было `?: string` — приводим к стилю проекта где optional дат — `null`).
- `workspace_id` denormalized комментарием отмечено как mock-only (хотя design doc писал «обязательный» — это не противоречит, я просто документирую что real backend вывел бы это).

### 5.3 API mock

```ts
api.listHandoffs(filter?: { since?: string }): Promise<HandoffList>
```

- `since` — ISO string lower bound по `created_at`. Дефолт — без ограничения (фильтр окна — UI-side, чтобы переключать без перезапроса).
- Filter cascade: `inCurrentWorkspace(handoff.from_agent_id)`.
- DevMode: empty → `[]`, loading → hang, error → throw.
- `await delay()` 120-380 ms как везде.

### 5.4 Что НЕ меняется (§6.4 design doc — подтверждено)

- `Agent` тип не получает новых полей.
- `gateway.yaml` не трогаем.
- Никакого «teams within team» / «squads» / «departments».

### Phase 3 — API mock

**Файл:** `src/prototype/lib/api.ts`

1. Импортировать `handoffs as fxHandoffs` из fixtures, тип `Handoff, HandoffList`.
2. Добавить метод `listHandoffs` рядом с другими `list*` (после `listAudit`):
   ```ts
   async listHandoffs(filter?: { since?: string }): Promise<HandoffList> {
     await delay()
     if (await _devGate() === 'empty') return { items: [], total: 0 }
     const all = fxHandoffs
       .filter(h => inCurrentWorkspace(h.from_agent_id))
       .filter(h => !filter?.since || h.created_at >= filter.since)
       .sort((a, b) => b.created_at.localeCompare(a.created_at))
     return { items: all, total: all.length }
   }
   ```

### Phase 5 — Canvas component

**Файл:** `src/prototype/components/team-map-canvas.tsx`

1. Props: `agents: Agent[]`, `handoffs: Handoff[]`, `positions: Record<string, {x,y}>`.
2. Виртуальный canvas 800×500. Контейнер `<div style={{position: 'relative', width: 800, height: 500}}>`.
3. SVG слой (поверх — `position: absolute; inset: 0; pointer-events: none для рёбер которые на нём`) рисует линии между центрами карточек. **Edge thickness:** базовая 2px, `+ 0.5px × (count-1)` для пары, cap на 6px.
4. HTML слой — карточки агентов как `<div>` абсолютно позиционированные, с аватаром/именем/ролью/статусом. Маленький компактный вариант, `~140×64px`.
5. **Aggregate edge state per pair:** если хотя бы 1 pending → класс `--pending` (пунктир, амбер); хотя бы 1 timed_out → `--stuck` (красноватый сплошной); иначе `--ok` (приглушённый сплошной); declined — отдельный тон или просто stuck.
6. CSS: добавить блок в `prototype.css` под `.prototype-root .team-map { … }`.

**Verify:** Build/lint clean. Граф рисуется визуально с 4 узлами и видимыми связями.

## 7. Risks (адаптировано из §11 design doc)

| Риск | Mitigation |
|---|---|
| Граф пуст в `ws_ops` после демо-данных (если перепутаю run_id) | Phase 2 — grep'нуть существующие runs в ws_ops перед seeding'ом, проверить что все 10 handoff'ов ссылаются на live run_id. |
| HoverCard прыгает при hover на тонкое ребро | Phase 6 — невидимая 8-10px hit-зона поверх видимого 2-6px stroke. |
| Static positions не масштабируются на разные viewport'ы | Canvas — fixed 800×500 с `overflow: auto` обёрткой. Для preview достаточно. Force-directed как enhancement если фича пройдёт. |
| `agent_call` step_type не learnable old runs (orphan) — никем не эмитится | Это ожидаемо. Тип добавляется future-proofing. Audit и Run Detail не падают, потому что `step_type` уже сейчас free-form string в audit. |
| Vocabulary дрейф — `node`, `edge` могут просочиться в copy | Phase 10 vocab check. В internal коде — норма (className, переменная), в visible text — нет. |
| Запись workspace_id на Handoff денормализована — backend будет иначе | Откомментировано в types.ts что это mock-only convenience. § 1.16 фиксирует правильную модель. |
